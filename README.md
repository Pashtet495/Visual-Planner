Версия 0.3.0
Обновления интерфейса и функционала:
- Не выполненные подзадачи можно выделить как избранные. Избранные задачи отображаются вверху списка не выполненных задач. Возможность выделить задачу как избранную можно в свойствах блока или в списке не выполненных задач.
- Добавлен блок заметок. Блок может быть любого размера. Имеет заголовок и основное тело.
- Добавлен функционал отмена/повторить (вернуть отмену) на глубину в 10 изменений. (стрелки вперёд и назад в хедере)
- Добавлено меню истории изменений. На последние 10 изменений. Можно сравнить разницу между текущим состоянием и выбранным из истории "снепшотом" (кнопка "{ }" в хедере)
- Добавлен встроенный AI агент, который может в паре с пользователем работать над блок схемами и Гантом.
- Для реализации работы с AI агентом добавлено сворачиваемое меню слева.
- В верхней части меню добавлена кнопка настройки подключения к нейросетям. 


!!!!!!!!!!!!!!!             В текущей версии необходима поддержка CORS со стороны сервера                !!!!!!!!!!!!!!!!!!!!!!!!!! 


 Поддерживается совместимые OpenAI API ендпойнты (для сервера LM Studio нужно вписать адрес http://127.0.0.1:1234/v1 поле ключ оставить пустым). (поле для выбора ембеддинг нейросети можно оставлять пустым, RAG встраивание пользовательской базы примеров пока не реализован).
- Для работы нейросети в программу внедрена система управления при помощи команд (примеры в исходном коде в системном промте агента полностью покрывают перечень добавленных команд, используйте примеры вместо документации) (в дальнейшем эти команды будут использоваться как tools для MCP).
- Агент "видит" изменения, которые делает пользователь. Для совместной работы "Пользователь+AI" не требуется ничего дополнительно делать.
- Агент кроме внесений изменений на схему или диаграмму Ганта может в чат писать пояснения к своей работе (отчитываться о том что он изменил) или давать ответы на запросы пользователя (допустим можно спросить у агента - выполнена ли какая-то задача, или какие задачи ещё не выполнены для начала работы над тем или иным этапом). Агент понимает контекст между сообщениями, то есть если вы обсуждали какую-то ветку блока, то не обязательно в дальнейшем диалоге уточнять про какую работу идёт речь.
- Добавлена функция "RepairJson". Проведён shit контроль слабыми нейросетями. Выявлены типичные синтаксические ошибки маленьких нейросетей. На основе частых ошибок реализована автопочинка невалидных команд. (но пока есть куда улучшать системный промт и синтаксис, в текущий момент не рекомендую использовать нейросети слабее Qwen3.6 35B A3B).

Не фикшенные баги:
- Кнопка "Stop" генерации - отключает генерацию для текущего рабочего пространства навсегда.
- Нет стабильности работы чата если переключать одну нейросеть на другую.
- Не оптимальны примеры в системном промте (сейчас они сформированы нейросетями).
- В некоторых браузерах при перетаскивании рабочего поля с зажатым курсором - выделяется часть текстового содержимого программы.



Планируемый функционал следующих изменений (если есть предложения - пишите. реализовать хотелки сразу проще, чем потом "вживлять" в готовые части программы).

- RAG встраивание. Пользовательские примеры. RAG поиск. Нейросеть сможет работать по соответствие пользовательским примерам. Сама искать похожие по типу задач ранее сделанные схемы и сразу адаптироваться под их внешний вид, подробность и структуру. Учитывать заданные в старых схемах временные отрезки на выполнение похожих задач, типичные подзадачи для больших задач и т.д.
- Финализация tools команд для создания полноценного MCP.
- Создание режима минимального интерфейса, когда программа сможет выдавать canvas подобное пространство со схемой для встраивания схем в сторонние проекты. Canvas подобная сущность + MCP с понятными для нейросетей Tools и компактно записанный пошаговый план всех работ и статус их выполнения вкупе с понятным визуальным представлением всей структуры проекта позволят одновременно и пользователю чётко понимать что делает нейросеть в сложных агенских задачах и самой нейросети использовать структурированную схему всех прошлых и будущих действий как долгосрочную память и потенциально более эффективно обращаться к другим инструментам.
- Добавление канбан интерфейса. Канбан карточки должны сами создаваться и раскладываться исходя из порядка связей для работ, статусов и подзадач, заданных в редакторе. Отслеживать простроченные задачи, исходя из даты в диаграммах Ганта и встроенного календаря проекта. Выделять задачи, которые "поедут" от просрочивания выбранной задач. Выделять связи между задачами, исходя из связей схемы.
- Добавление вложений к задачам и подзадачам. Прикрепление вложений (файлов) позволит быстро находить и открывать пользователю файлы/документы связанные с требованиями к задаче или к документам, закрывающим задачу. Вложения напрямую будут указывать AI агенту какие документы нужно просматривать в случае вопросов по задаче. Это существенно уменьшит ошибки мусорного контекста в RAG встраивании, будет экономить контекст нейросети. Увеличит чёткость понимания задачи для нейросети, которая будет работать со схемами в качестве долгосрочной памяти.



Версия 0.2.7
ВНИМАНИЕ! Сохранения от прошлых версий могут работать не корректно.

Список изменений:
- Если выделено несколько блоков, то копируются все выделенные блоки
- При копировании блоков есть минимальная дальность не достигая которую блоки не копируются (защита от случайного перетаскивания при выделении блоков)
- Шрифт в блоках уменьшен, добавлен перенос текста в названиях блоков (для блоков с длинными названиями)
- Добавлены модальные окна для создание связей и управления принадлежностью сабграфам при работе через контекстное меню
- При редактировании блоков через меню с шестирёнкой клавиша DEL удаляла блок вместо редактируемого текста.
- Новый метод создания блоков: вытягивание связи в пустую область. Блок создаётся в месте где пользователь отпускает мышку. Открывается модальное окно для создания блока и ввода его параметров. Если пользователь случайно вытянул связь или передумал - связь нужно навести обратно на блок. Блок получит широкую красную подсветку, обозначающую отмену действия.
- Внедрён поиск. Поиск применяется к названию блоков, описаниям, задачам в to-do списках блоков.
- В правом меню добавлен список всех не выполненных задач текущего рабочего пространтсва с возможностью подсветить блоки с не выполненными задачами. Через это меню можно отмечать выполнение задач. Меню сворачиваемое.
- Правое меню теперь можно сворачивать и регулировать его ширину.
- Обновлён метод выделения активных блоков. Теперь подсвечиаются и рамки и сердцевина блока для лучшего обозначения того что выделено. Для выделения блок из меню не выполненных задач добавлена функция автоматического центрирования рабочей области на выделенном блоке.
- Для размещения строки поиска оптимизирован хеддер. Кнопки "создать", "Сохранить", "загрузить" переделаны в иконки "листочек", "Дискета", "Открытая папка".
- Новый формат отображения выполненных задач в To-do списках (перечёркивание заменено на зелёный цвет).
- Теперь задачи в To-do списках можно трансформировать в дочерние подзадачи, отображаемые в диаграмме Ганта внутри материнского блока. Для этого добавлена новая опция для списка to-do задач. Для каждой задачи можно выставить галочку о добавлении подзадачи и при этом активируется настрйока промежутка выполнения задачи внутри материнской задачи. To-do подзадачи в диаграмме ганта отображаются в скрываемых частях материнской задачи. В случае выполнения подзадачи они автоматически перекрашиваются в зелёный цвет. Период выполнения подзадачи внутри мтаринской можно менять как в ранее описанном меню свойств, так и в диаграмме Ганта, вытягивая края. В случае если период дочерней задчи вытягивается шире периода материнской задачи - линия материнской задачи автоматически вытягивается вслед за дочерней. В случае  перемещения материнской задачи по таймлайну - дочерние задачи перемещаются вместе с материнской.
- Оптимизировано lson сохранение для минимизации инструкций для генерации схем при помощи нейросетей. Уменьшено количество обязательных параметров. Программа сама восстанавливает недостающие параметры, заменяя их на дефолтные значения.
Пример сохранения простой схемы:

  "nodes": [
    {"id": "n1","name": "Начало","shape": "rect","x": 750,"y": 340},

Опиасние изменений версии 0.1
Новые функции:
- Сохранения можно добавить на рабочие пространства как новый сабграф (сложные проекты можно собирать по частям, а потом их сохранения добавлять в общий проект)
- Добавлена функция выставления меток на таймлайн диаграммы Ганта при помощи кнопки "закладки"
- Добавлена инструкция при первом запуске

Оптимизация интерфейса:
- Переработана система выделения элементов цветовыми маркерами
- Добавлена возможность отключать отображение отдельных блоков в диаграммах Ганта
- В диаграммах Ганта улучшен визуал демонстрации связей (ранее при выделении блока в рабочем пространстве или линии в диаграммах подсвечивались связанные блоки, теперь кроме этого в диаграмме Ганта рисуются стрелки связей между зависимыми блоками при их выделении)
- Улучшен интерфейс корректирования положений элементов в диаграмме Ганта (курсор меняется на вертикальную двустороннюю стрелку в местах где можно менять линии в диаграмме между собой выше/ниже, курсор меняется на горизонтальную двустороннюю стрелку в местах где можно расятнуть линию по таймлайну, действия по изменениям подкрепляются визуальными изменениями при их совершении)
- оптимизирован процесс переименования блоков при их создании
- Меню информации о блоках (знак i в круг рядом с блоком) заменён на меню редактирования блоков (знак шестерни в кружке). В быстром меню можно переименовать блок, изменить описания, создавать задачи для блока, отмечать задачи, удалять задачи, менять цветовую маркировку блока

Устранение багов:
- При создании связей конец связи смещался от курсора
- Прилипание связи при её создании происходило со смещением относительно положения блока
- Для блоков не прямоугольной формы стрелки связей в некоторых положения проваливались внутри блока.
